Unity适配解决方案

发表于 2019-11-29 |

一、适配方案设想

​ 对于各种分辨率的手机，适配是一个头疼的事情，所以只能一点点探索。

​ 对于一个好的适配，我们需要什么？：

  1. 控件：要保证控件在所有分辨率下都能显示在屏幕可操作区域，例如：刘海屏不能把可点击区域放置在刘海屏里
  2. 背景图：在宽高比矮胖（ipad）和 高瘦（iphone X） 上都可以显示正常

​ 对应方案：

​ 出于目前大部分手机宽高比都是0.56（iPhone6/7/8）或者 0.46 (iPhoneX/XR/XS) ，所以在美术出图时我们要求设计分辨率是基于750 * 1634 ，并且保证

控件布局在压缩 到 750 *1334 下也能正常显示。

​ 而程序上，要完成上面的需求，做法是控件和背景图的适配需要分开处理，这样可以单独控制内容层和背景层的缩放与位置。

​ 首先对画布canvas的控制来满足不同分辨率可以

二、canvas 和 canvas Scale 的配置选择

  • Render Mode (渲染模式)： 选择Screen Space -Camera (相机的作用就是把它所投射获取到的界面当做UI界面)

​ 预制体上添加canvas Scale组件来控制画布适配

  • 根据参考分辨率和当前设备分辨率，得出一个“合适”的scaleFactor。至于设备分辨率和参考分辨率比例不同，这个问题交由Canvas自动设置width/height来解决。
  • Screen Match Mode : 选择Expand scaleFactor = Mathf.Min(screenSize.x / m_ReferenceResolution.x, screenSize.y / m_ReferenceResolution.y) 结果是尽量把UI在设备屏幕给放大
  • Reference Resolution ： 填上我们的参考分配率 750*1334（iPhone 6/7/8）

以上便是本项目分辨率适配的方案，每个预制体上都挂载上这两个组件后，便可以自动扩张拉伸长宽比缩放比较小的部分，将缺少的部分拉伸

三、代码控制 canvas 和 canvas Scale

​ 当然美术、策划、程序在预制体制作时都要加这2个组件、并且设置正确的话会是个很头疼的问题。

​ 所以我们选择在代码里修改，这样预制体制作时可以直接添加这两个组件不需要管具体设置，在每个界面打开的时候，如果有canvas和canvaseScale组件就对这两个组件进行强制设置，而页面的加载是底层的，只需要在加载页面时获取这两个组件即可

四、 内容层和背景层各自分开适配处理

  1. 刘海屏
    为了照顾刘海屏， 所以控件层需要顶部下移60px，目前用了对称拉伸的方式所以做法是上下各自移动 60px ，上面留给刘海，用背景图来填充。（这里有一个遗憾的地方是没有办法判断这个手机是刘海萍还是全面屏，只能暂时通过宽高比低于0.56就把控件节点上下移动60像素）
    需要进行控件区域适配的界面在init()的时候调用UICanvas的initCanvas函数，传入对应的_contentNode节点就可以修改区域的Top和Buttom
  1. iPad矮胖界面的适配处理
    对于iPad这种宽高比比较大的机型来说，如果直接拉宽背景图可能会很丑，这时候我们有两种方案
    (1) 等比拉伸背景图（优点：处理简单  缺点： 这样会导致部分区域看不到，美观可能有影响）
(2) 单独做一个左右区域的预制体，判断宽高比过高时才加载这个预制体（优点： 界面可以很和谐美观  缺点：需要每个预制体都独立制作，并且需要考虑资源是否只打某个平台的包进行管理）
